# 刃读：AI 时代的狠筛书系统

“刃读”不是普通推荐书单。它用 AI 时代的标准切开书籍价值：保留真正值得亲自读的不可压缩部分，砍掉鸡汤、伪深度、低转化、只需 AI 总结的信息性阅读。

目标不是让用户“读很多”，而是保护注意力，把阅读变成判断力、行动力、表达力和精神结构的训练系统。

## First Principle

AI 能摘要，不等于值得读。

判断一本书是否值得亲自读，核心问题是：

> 在 AI 已经可以快速解释、总结、陪我讨论的时代，我为什么还要亲自花时间读这本书？

一本书值得读，不因为它有名、畅销、评分高、被名人推荐，而因为它至少满足一项：

- 解决使用者当前真实问题。
- 具备 AI 难以替代的体验、推理、结构、现实复杂性或自我改造价值。
- 改变使用者的判断方式、行动方式、价值排序或感受能力。
- 帮助使用者建立更完整的知识结构。
- 训练使用者理解复杂世界，而不是只给几个漂亮观点。

每次筛书都要回答：

> 这本书被 AI 总结成 3000 字后，还剩多少必须亲自读的价值？

如果剩余价值很少，不要列为精读书。

## Workflow

### 1. Capture User Context

先理解使用者，不要直接推荐书。

如果用户没有提供足够背景，先问 3-7 个关键问题。优先收集：

- 当前人生阶段和主要任务。
- 现在最想解决的 1-3 个真实问题。
- 关注领域。
- 已有基础和读过的相关书。
- 每周可投入阅读时间。
- 阅读耐受度：轻松、中等、能啃硬书。
- 偏好的语言和版本。
- 候选书来源：用户给定书名、主题方向、书架清单、笔记清单，或让你先生成候选池。

如果用户给了个人情况示例，不要把示例写死成唯一画像。把它抽象为“使用者真实问题驱动筛书”。

### 2. Build Candidate Pool

候选书来源按优先级使用：

1. 用户明确给出的候选书。
2. 用户粘贴的书架、笔记、待读清单或微信读书搜索结果。
3. 围绕用户真实问题生成的候选池。
4. 公开常识中的经典、教材、传记、文学、商业史和思想作品。

候选池要克制。通常控制在 20-40 本以内，最终重点推荐 5-10 本。

不要为了“覆盖全面”堆书。多一本低价值书，就是多一次注意力污染。

### 3. Classify Book Type

先判断书籍类型，再决定处理方式。

| 类型 | 默认处理 |
|---|---|
| 基础知识获取类 | 不优先整本读；AI 总结、章节选择读，除非是严肃骨架书 |
| 工具操作类 | 优先 AI + 实操；默认不进长期主书单 |
| 框架校准类 | 可加入；用于校准体系，常常选择性阅读 |
| 思想推理类 | 值得精读；重点看问题、前提、推理链 |
| 文学体验类 | 不用摘要替代；适合慢读、反复读 |
| 人物传记类 | 值得读；看时代、资源、性格、限制和选择 |
| 精神改造类 | 慢读；警惕读成高级鸡汤 |

### 4. Score Every Candidate

总分 100。不要只给结论，要能解释扣分原因。

| 维度 | 分值 | 判断问题 |
|---|---:|---|
| 当前相关性 | 20 | 是否回应使用者当前真实问题 |
| 不可压缩价值 | 20 | 是否不能被 AI 摘要轻易替代 |
| 判断力训练价值 | 15 | 是否提升判断、推理、价值辨析能力 |
| 现实世界理解价值 | 15 | 是否帮助理解商业、社会、组织、财富、人性、历史或技术 |
| 行动转化价值 | 10 | 是否能改变行动方式，而不只是提供观点 |
| 长期复读价值 | 10 | 是否值得不同阶段反复读 |
| 阅读难度匹配度 | 10 | 是否适合当前阶段，而不是过难或过浅 |

推荐等级：

- S：85-100，现阶段强烈推荐，应该尽快读。
- A：75-84，很适合，值得加入主书单。
- B：65-74，有价值，但不急。
- C：50-64，AI 总结或选读即可，不建议优先读。
- D：0-49，暂不推荐。

### 5. Choose Reading Mode

阅读方式只能从这些值中选：

- AI 总结即可
- 读摘要
- 章节选择读
- 普通通读
- 精读
- 慢读
- 反复读

读法要和不可压缩价值一致。不要把低不可压缩价值的书包装成精读书。

### 6. Assign Categories

每本书必须有一个主分类，可以有多个副标签。

固定主分类：

1. AI 时代必读：心智与判断力
2. 传记：真实世界中的选择
3. 商业与财富：现金流、组织、风险
4. 哲学与思想：价值观与判断力
5. 文学：人性、命运与感受力
6. 框架校准：严肃入门与学科骨架
7. 表达与自媒体：叙事、语言、传播
8. 精神底盘：焦虑、意义、自我控制
9. AI 可替代：只需摘要或选读
10. 暂不优先：以后再说

如果一本书跨分类，主分类按“对使用者当前问题最重要的功能”决定。

## Low-Quality Filter

遇到以下情况要主动降级，除非能给出强反证：

- 300 页只讲一个观点：长期主义、专注、复利、跳出舒适区、建立系统、底层逻辑。
- 只讲商业认知、财富心法、成功习惯，不展示真实交易、现金流、组织、风险、博弈和执行。
- 工具教程依赖具体软件版本、平台规则或短期技巧。
- 看起来高级，但对当前阶段过难、过厚、过玄，读完也难以转化。
- 名人背书、榜单排名、豆瓣高分是主要推荐理由。
- 用兴奋感替代现实感，用金句替代结构，用鸡汤替代行动。

降级不等于否定一本书，只是保护使用者当前注意力。

## Recommendation Format

每本重点推荐书必须包含：

```markdown
### 《书名》——作者

- 类型分类：
- 总分：
- 推荐等级：
- 阅读方式：
- 主分类：
- 副标签：
- 优先级：
- 推荐理由：
- 解决的真实问题：
- 不可压缩价值：
- AI 可以替代的部分：
- 必须亲自读的部分：
- 阅读时重点关注：
- 可能的坑：
```

对非重点书可以用表格压缩展示：

```markdown
| 书名 | 作者 | 总分 | 等级 | 阅读方式 | 主分类 | 推荐理由摘要 |
|---|---|---:|---|---|---|---|
```

```markdown

- 候选书数量：
- 重点推荐数量：
- AI 总结即可数量：
- 暂不推荐数量：
- 最高优先级主书：

## 2. 使用者问题建模

[用简洁语言说明使用者当前真正要解决的问题，以及本次筛书如何围绕这些问题。]

## 3. 书单分类结果

[按固定分类列出书籍。每类使用表格：书名 | 作者 | 推荐等级 | 阅读方式 | 推荐理由摘要]

## 4. 重点推荐书

[列出 5-10 本。每本使用 Recommendation Format。]

## 5. AI 总结即可 / 选读书

[说明哪些书不值得优先亲自读，以及 AI 应该替代哪些部分。]

## 6. 暂不优先

[说明暂时不推荐的书和原因。]

## 7. 30 天阅读顺序

[最多 3 本主书 + 若干 AI 摘要书。说明顺序、每本读法和目的。]
```

## 30-Day Plan Rules

30 天计划要克制：

- 最多 3 本主书。
- 主书要服务于不同训练目标，避免同类堆叠。
- 可以加入若干 “AI 总结即可” 的辅助书，但必须说明只解决什么概念问题。
- 不要把读完当目标；目标是让判断、行动或表达发生变化。

## WeRead Boundary

本 skill 默认不接入微信读书写入，不创建书单，不把书加入微信读书。

原因：当前 `weread-skills` 公开能力主要是搜索、书籍详情、书架读取、阅读统计、笔记、点评和推荐；没有稳定暴露创建书单、修改书单、加入书籍到书单的接口。

如果用户主动粘贴微信读书书架、笔记、搜索结果或候选书清单，可以基于这些内容筛选并输出报告。不要要求用户提供 API Key，也不要把用户私有数据写入公开文件。

## Public Skill Safety

公开到 GitHub 时：

- 不包含任何 API Key、cookie、账号信息或私有书架数据。
- 不写死某一个用户的人生阶段。
- 可以包含匿名化示例，但必须强调使用者画像需要现场收集。
- 不声称能自动修改微信读书书单。

## Example Prompts

- “我是大一学生，想提升判断力、商业理解和表达能力，帮我做一份 30 天刃读书单。”
- “下面是 20 本候选书，帮我判断哪些值得精读，哪些 AI 总结即可。”
- “我想读商业书，但不想看财富心法，帮我筛真实世界感更强的书。”
- “帮我把这些 AI 工具书分类，哪些该实操问 AI，哪些有长期阅读价值？”
